Expectation-Maximization Algorithm

The Expectation-Maximization Algorithm (EM) aims to help us estimate the m/u probabilities for all fields being used for matching. It does this with an initial guessing step, then a re-estimation step that repeats indefinitely until the numbers stabilize.


Step 1 - Start with an initial guess

The algorithm is given ballparked starting values for each field. This might look something like this:

  • Name: m = 0.90, u = 0.001
  • ZIP: m = 0.55, u = 0.005
  • Gender: m = 0.99, u = 0.5

In reference to the term "ballpark", these numbers do not need to be perfect. Technically, you could make these numbers anything you want, but it will take substantially longer to compute. Generally, human intuition is good enough for a starting point here.


Step 2 - Expectation

Using the current values, we score every pair and estimate: what is the probability this pair is truly a match? After this step, every pair will get a soft label rather than a hard yes/no:

  • Pair A might be 0.99
  • Pair B might be 0.17
  • Pair C might be 0.002

This step is called Expectation because all we are doing is computing the expected match status of each pair under our current model.


Step 3 - Maximization

Now, we treat the estimated match probabilities we just calculated as weights and recompute the m/u values. For example, if name agreement appears mostly in pairs that have high probability of being true matches, the estimated M value for name stays high. Or, if ZIP agreement appears frequently even among pairs with low match probability, the estimated U value for ZIP will rise. Continually applying this step will better fit our actual data.


Step 4 - Repeat

Our EM algorithm will repeat steps 2 and 3 until we start approaching convergence, where the numbers are only changing by 0.01 or less every re-run. Once we reach that point, we have successfully estimated our m/u probability for every field used in our GRR.
